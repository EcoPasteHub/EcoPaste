# EcoPaste 分组 UI 重建 TODO

> 本文件只记录后续实施路线，不描述当前现状。
> 默认前提：相关能力尚未落地，后续按零状态逐步接入。

## 0. 目标

把“自定义分组”相关 UI 收拢成一套可复用的基础能力，避免主窗口、偏好设置、菜单、弹框各写一套。

核心目标：

- 统一 Dropdown 的默认图标尺寸、交互外观和可覆写方式。
- 统一分组图标渲染，兼容 lets-icons 预设图标与自定义 SVG。
- 统一分组新增 / 编辑弹框，主窗口和偏好设置共用同一实现。
- 统一分组管理入口、右键菜单、更多菜单和新增入口的行为。
- 保持“只显示图标，文字交给 Tooltip / Menu / KeyHint”的交互风格。

## 1. 范围

### 1.1 计划覆盖

- 主窗口的分组区域
- 主窗口的更多 / 新增入口
- 分组右键菜单
- 偏好设置里的分组管理页
- 分组新增 / 编辑弹框
- 分组图标选择与上传
- 统一 Dropdown 组件

### 1.2 暂不展开

- 条目自动归组
- 批量移动条目到分组
- 分组筛选逻辑重写
- 任何与剪贴板数据流无关的业务改造

## 2. 数据与约束

### 2.1 分组数据目标形态

分组后续应至少支持这些字段：

- `id`
- `name`
- `icon`
- `isHidden`
- `sortOrder`
- `createdAt`
- `updatedAt`

其中：

- `icon` 需要兼容预设图标和自定义 SVG。
- `isHidden` 用于控制分组是否在主界面可见。
- `createdAt` / `updatedAt` 只作为分组自身元数据，不混入别的语义。

### 2.2 交互原则

- 主要按钮只显示图标。
- 说明文字放 Tooltip、Menu、KeyHint 或弹框标题里。
- 同类图标尺寸要统一，默认尺寸收口到组件内部。
- 特殊场景可以通过单独参数覆盖。
- 主窗口与偏好设置尽量复用同一套组件，不复制逻辑。

## 3. 分阶段推进

### 3.1 第一阶段：Dropdown 基础组件

先抽出一个统一的 `Dropdown` 组件，作为所有业务菜单的默认入口。

要做的事：

- [x] 统一菜单项图标默认尺寸。
- [x] 特殊图标样式通过 `icon` 传入自定义 ReactNode 覆盖。
- [x] 保留 antd 原生能力，不压缩成只能应付一种场景的薄壳。
- [x] 让所有使用 Dropdown 的地方逐步替换到这个新组件。

验收标准：

- 菜单图标尺寸默认一致。
- 特殊菜单可以单独调大或调小。
- 组件命名和目录结构与现有 `Tooltip` 等基础组件风格一致。

### 3.2 第二阶段：分组图标基础能力

提炼一层统一的分组图标展示能力。

要做的事：

- [x] 支持 lets-icons 预设图标。
- [x] 支持自定义 SVG 图标。
- [x] 统一图标大小、对齐和间距。
- [x] 统一选择态与禁用态样式。
- [x] 避免同一图标在列表、菜单、弹框里显示出不同尺寸。

验收标准：

- 同一个分组图标在各处显示一致。
- 预设图标与自定义 SVG 的显示方式一致。
- 图标字符或类名只在一个地方收口。

### 3.3 第三阶段：共享分组 Modal

把分组新增 / 编辑弹框抽成一个共享 Modal。

要做的事：

- [x] 用同一个弹框覆盖“新增”和“编辑”两种模式。
- [x] 名称字段走 `required` 校验。
- [x] 自动聚焦到名称输入框。
- [x] 图标选择提供预设图标与自定义 SVG 上传。
- [x] 自定义 SVG 采用按钮 + Modal 的方式处理。
- [x] 选择文件时使用 Tauri dialog，dialog 打开时窗口无法失焦关闭。
- [x] 图标上传区域下方提供下载说明链接，并可直接点击。
- [x] 删除图标、预设图标切换、重新上传等状态都在同一弹框内解决。

验收标准：

- 新增和编辑只有一份实现。
- 进入弹框后，名称必填且自动聚焦。
- 图标选择、上传、删除流程完整。
- 主窗口和偏好设置都能直接复用。

### 3.4 第四阶段：主窗口接入

把主窗口的分组相关入口切到统一组件。

要做的事：

- [ ] 分组栏改用统一 Dropdown。
- [ ] 更多按钮、+ 按钮、右键菜单统一走同一套菜单模型。
- [ ] 菜单里的管理、编辑、新增等入口只保留一份定义。
- [x] 分组条目图标切到统一渲染方式。
- [x] 新建 / 编辑分组时复用共享 Modal。

验收标准：

- 主窗口的分组 UI 不再依赖局部拼装的重复逻辑。
- 菜单、图标、弹框的视觉一致。
- 新增 / 编辑入口都能正常工作。

### 3.5 第五阶段：偏好设置接入

把偏好设置里的分组管理改成同一套基础能力。

要做的事：

- 将分组管理入口放到合适的设置分组中。
- 复用统一 Dropdown / Modal / 图标组件。
- 管理页只负责编排，不再复制业务逻辑。
- 保存后能推动主窗口同步刷新。

验收标准：

- 偏好设置里的分组管理与主窗口保持同一套规则。
- 保存后主窗口能即时看到变化。
- 管理页自身不再维护一套独立实现。

### 3.6 第六阶段：目录与命名收口

把公共组件整理到更贴近现有项目风格的位置。

要做的事：

- 新公共组件命名直接用 `Dropdown`，不再另起容易混淆的名字。
- 组件文件夹结构与 `Tooltip` 等基础组件保持一致。
- 通用逻辑集中在基础组件，页面组件只做组合。

验收标准：

- 新增组件不会散落成多个临时文件。
- 命名清晰，查找路径稳定。

## 4. 实施顺序建议

1. 先落 `Dropdown` 基础组件。
2. 再做分组图标能力。
3. 再抽共享分组 Modal。
4. 然后替换主窗口的入口。
5. 再接偏好设置。
6. 最后整理目录和命名。

## 5. 完成标准

- 主窗口与偏好设置共享同一套分组基础能力。
- Dropdown 菜单图标大小统一。
- 分组图标统一渲染。
- 新增 / 编辑弹框统一。
- 只保留一套可维护的实现路径。
