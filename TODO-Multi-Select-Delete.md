# TODO: 列表多选与批量删除

## 目标

在主剪贴板列表中加入多选模式，支持批量选择、确认删除、退出多选，并让删除行为走 Rust 命令与统一事件刷新。

## 范围

- Rust 仓储层新增批量删除能力，复用单条删除的资源清理逻辑。
- Tauri command 暴露 `delete_clipboard_items(ids)`，命令层保持薄。
- 前端列表进入多选模式后，行点击改为切换选中，不触发复制或粘贴。
- 工具栏提供全选、反选、删除选中、退出。
- 删除前使用 Ant Design `Modal` 二次确认。
- 列表聚焦时支持 `Cmd/Ctrl+A` 全选，`Esc` 退出多选。

## 实施要点

- `db/items.rs` 增加 `delete_items_bulk(pool, ids)`，事务包裹，返回删除数量。
- 若当前删除仍是硬删除，资源清理 helper 要与单条删除共用；若先落地回收站，则本任务应改为批量移入回收站。
- `stores/clipboardView.ts` 增加 `selectionMode` 与 `selectedIds`，并提供显式 action。
- 多选模式下卡片左侧显示 checkbox，键盘高亮可以继续保留单选游标。
- 删除成功后清空选区并退出多选，列表根据事件或本地更新刷新。

## 验收

- 可连续选择、取消选择、全选、反选并批量删除。
- 多选模式不会误触发粘贴、复制、预览或行点击自动粘贴。
- 删除失败有日志和用户可见提示，成功后列表、页脚统计和当前筛选状态一致。
- macOS 与 Windows 主窗口键盘路径都可用。
